💡 어디에 영향을 주나요?
관찰 내용 생성 모드에만 적용되어 문장의 표현 톤을 조절합니다.
· A: 심층·독창·탁월함이 드러나도록   · C: 핵심 내용을 균형 있게   · E: 사실 기반으로 미흡함도 포함
채점 모드에서는 편향 방지를 위해 AI에 전달되지 않습니다 (루브릭만으로 순수 채점).
🎤 발표 관찰 체크리스트 선택

발표 현장에서 교사가 관찰한 내용을 체크하면 채점 시 발표력 관련 항목에 반영됩니다.

🤝 협업 능력 체크리스트 선택

학생 결과물만으로는 판단하기 어려운 협업 능력을 교사가 체크하면 채점 시 협업 관련 항목에 반영됩니다.

📋 채점 기준 (루브릭)

💡 채점 모드 팁: "A 20점 B 17점 C 14점 D 11점 E 8점" 형태로 등급별 점수를 명시하면 AI가 그 값만 사용합니다. 중간 값(19·18점 등)은 자동으로 가까운 허용 값으로 보정됩니다.

💡 사용 방법
  1. 실행 모드를 선택하세요 — 관찰 내용 생성 · 루브릭 채점 · 동시 실행(추천)
  2. AI 제공자(Claude·ChatGPT·Gemini)를 선택하고 API 키와 과목명을 입력하세요
  3. 채점 기준(루브릭)을 입력하거나 파일로 불러오세요
    관찰 모드에선 선택 사항 · 채점 모드에선 권장
  4. 필요 시 발표 관찰 · 협업 능력 체크리스트를 펼쳐서 입력하세요 (채점 시 해당 루브릭 항목에 반영)
  5. 학생 수행평가 파일(PDF·DOCX·PPTX·JPG·PNG)을 업로드하세요 (파일명 규칙은 아래 참고)
  6. 수행평가 분석 시작 버튼 클릭
  7. 결과 카드의 📋 복사 클릭 → 생기부 작성 도우미에 붙여넣기 · 채점 결과는 💾 CSV 저장으로 성적부 활용
🎯 실행 모드 안내
  • 관찰 내용 생성 — 생기부 세특용 "~함." 문장 3버전 생성
  • 루브릭 채점 — 평가 항목별 점수·등급·근거 + CSV 저장
  • 동시 실행 (추천) — 파일 파싱·추출 1회로 관찰+채점 동시 처리 (비용 최소화)
📁 파일명 규칙
예시 의미
홍길동.pdf 단일 학생 · 단일 파일
홍길동__보고서.pdf
홍길동__발표.pptx		 🔗 홍길동 1명 — 2개 파일 통합 분석
언더바 2개(__)로 학생 이름과 파일 설명을 구분하면, 같은 학생 이름을 가진 파일들이 자동으로 묶여 하나의 관찰 내용으로 생성됩니다. (API 비용 절감 ✓)
1모둠-홍길동,김철수,이영희.pdf 모둠 활동 (이름을 쉼표로 구분)
1모둠-홍길동,김철수__보고서.pdf
1모둠-홍길동,김철수__발표.pptx		 🔗 모둠 + 통합 분석 조합
📌 출력 형식
생기부 작성 도우미가 요구하는 "~함." 종결형 관찰 문장을 줄바꿈으로 나열합니다.
각 문장에 단원명 + 구체적 활동 + 학생의 주장·분석·결론을 압축하여
도우미가 고품질 생기부 문장을 생성할 수 있도록 최적화합니다.
